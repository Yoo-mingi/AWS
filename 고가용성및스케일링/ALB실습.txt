데이터가 전송될 EC2인스턴스 생성. one,two라 가정.
키 페어가 필요없다면(SSH를 안쓴다면)안만 들어도 ㅇㅋ

보안그룹으로 http통신 open

원하는 것 : 하나의 url(ALB)로 2개의 인스턴스에 부하분산.

ALB 생성.
이름을 입력하고 가용영역 선택.
보안그룹 할당. 없다면 만드는데, http트래픽 on으로 설정

대상그룹 생성 : 
인스턴스를 묶을수도있고 다른것도 가능.
타겟그룹이름 설정.
프로토콜과 port(http,80)설정.
타겟그룹에 들어갈 인스턴스 클릭

타겟그룹을 만들고 ALB로 돌아가 리슨 앤 라우트 에서 타겟그룹 선택.
그리고 Create 클릭.

그럼 생성된 로드밸런서가 dns이름을 뱉을탠대 해당 이름을 사용하면 alb로 접속.

*******

네트워크 보안 : 보안그룹을 통해 alb에 엑세스 하고 있다.
ec2인스턴스는 EC2보안그룹을 통해 접속이 가능하다.

인스턴스에 시큐리티 그룹으로 들어간다.
그리고 EC2의 인바운드 그룹을 편집한다.
http 는 맞지만 IP를 0/0이 아니라 시큐리티 그룹의 ALB를 선택하는 것이다.
그럼 EC2로 바로 접속은 불가능하지만 ALB를 통한 접근은 가능하게 된다.
