Elastic file System
관리형 파일 시스템이며 많은 EC2인스턴스에 연결가능하고 다른 AZ에도 연결가능.
EFS고 보안그룹으로 둘러싸고 다른 가용영역에 EC2가 있다해도 같은 EFS에 연결가능.
내부적으로는 EFS프로토콜호환되며 윈ㄷ우가 아닌 리눅스와 호환.

미리 용량 계획이 필요없음. 자동으로 확장되며 사용량에 따라 비용지불
EFS스케일은 NFS클라이언트 수천개와 10기가 이상의 처리량 확보 가능
자동 확장가능


처리량 모드에 여러개가 있음.
버스팅 : 50M에 100M을 추가
프로비젼 : 스토리지 크기에 상관없이 처리량을 설정하고 싶을때 사용.
엘라스틱 : 워크로드에 따라 처리량을 자동으로 저장. 예를 들어 읽기는 3기가, 쓰는 1기가 처럼.

스토리지클래스 : 
스토리지티어 : 며칠 후 파일을 다른 계층으로 이동가능. 자주 액세스 하는 계층에서 사용하지 않는다면 자주 사용하지 않는 계층(EFS-IA)에 저장해 비용을 감소시키는 것.
수명주기를 정해야한다. 어떤 파일을 특정일(예로들자면60일)사용하지 않는다면 EFS IA로 보내는 것이다.

one zone EFS : 하나의 AZ에만 있고 백업이 가능하며 액세스 빈도가 낮은 스토리지 계층과 호환되지 않음.
따라서 원존IA라 불리며 할인율이 크다.
